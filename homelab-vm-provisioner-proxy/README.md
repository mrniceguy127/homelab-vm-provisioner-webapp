# Homelab VM Provisioner Proxy

Dead-simple reverse proxy that serves the React client and proxies API requests to the backend.

## Purpose

The proxy is the single public-facing entry point for the entire application:
- Serves static React client files from `public/`
- Proxies API requests to the Express backend
- Eliminates CORS issues by hosting frontend and backend under same origin
- Provides clean URL structure without exposing backend port

## Architecture

```
Browser → Proxy (port 3000) → API (port 3001)
         ↓
      Static Files (React app)
```

The proxy only serves static files and forwards `/api` and `/health` requests to the
API. The API handles provisioning, the RabbitMQ job queue, the database service, and
the worker; those components are not visible to the proxy. See the
[workspace README](../README.md) for the full architecture.

## How It Works

1. **Static Files**: All requests are first checked against `public/` directory
2. **API Routes**: Requests to `/api/*` and `/health` are proxied to backend
3. **Health Check**: Requests to `/proxy-health` return proxy status (not proxied)
4. **SPA Fallback**: All other requests serve `public/index.html` (React Router support)

## Configuration

Environment variables:

- `PORT` - Proxy server port (default: `3000`)
- `API_URL` - Backend API URL (default: `http://localhost:3001`)

Create `.env` file:

```bash
PORT=3000
API_URL=http://localhost:3001
```

## Usage

### Local Development

```bash

npm install

# Start proxy (assumes API is running on port 3001)
npm start

# Start with custom configuration
PORT=8080 API_URL=http://api.example.com npm start
```

### Docker Deployment

```bash
# Build Docker image
./build

# Start container
./start

# Custom ports
PROXY_PORT=8080 ./start
```

## Endpoints

| Path | Behavior |
|------|----------|
| `GET /proxy-health` | Proxy health check (not proxied) |
| `GET /health` | Proxied to backend API |
| `/api/*` | Proxied to backend API |
| `/*` | Served from static files (React SPA) |

### Examples

```bash
# Proxy health (returns immediately)
curl http://localhost:3000/proxy-health
# {"status":"ok"}

# Backend health (proxied)
curl http://localhost:3000/health
# {"ok":true}

# API request (proxied)
curl http://localhost:3000/api/vms
# {...vms data...}

# React app (served from public/)
curl http://localhost:3000/
# <!DOCTYPE html>...React app HTML...
```

## Static Files

The proxy expects static files in `public/`:

```text
public/
├── index.html        # React app entry point
├── assets/          # Vite-generated bundles
│   ├── index-[hash].js
│   └── index-[hash].css
└── vite.svg         # App icon
```

Static files are generated by:
1. Building the client: `cd ../homelab-vm-provisioner-client && npm run build`
2. Copying to proxy: `cp -r ../homelab-vm-provisioner-client/dist/* public/`

Or use the monorepo build script: `./build` from workspace root.

## Proxy Implementation

The proxy uses `http-proxy-middleware` for request forwarding:

```javascript
import { createProxyMiddleware } from 'http-proxy-middleware';

// Proxy configuration
const apiProxy = createProxyMiddleware({
  target: process.env.API_URL || 'http://localhost:3001',
  changeOrigin: true,
  logLevel: 'warn',
});

// Mount proxy middleware
app.use('/health', apiProxy);
app.use('/api', apiProxy);

// Serve static files
app.use(express.static('public'));

// SPA fallback
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, '../public/index.html'));
});
```

## Docker


Pass environment variables to container:

```bash
docker run -d \
  -p 8080:8080 \
  -e PORT=8080 \
  -e API_URL=http://api.example.com \
  --name hlvmp-proxy \
  homelab-vm-provisioner-proxy
```

### Volume Mounts

For development, mount static files as a volume:

```bash
docker run -d \
  -p 3000:3000 \
  -v $(pwd)/public:/app/public \
  --name hlvmp-proxy \
  homelab-vm-provisioner-proxy
```

## Development

The proxy is intentionally minimal - no tests, no build step, just a straightforward Express server with http-proxy-middleware.

Key files:
- `src/server.js`: Express server with proxy middleware and static file serving
- `package.json`: Dependencies (express, http-proxy-middleware)

For local development:
1. Start the API on port 3001: `cd ../homelab-vm-provisioner-api && PORT=3001 npm start`
2. Build the client: `cd ../homelab-vm-provisioner-client && npm run build`
3. Copy client files: `cp -r ../homelab-vm-provisioner-client/dist/* public/`
4. Start the proxy: `npm start`
5. Access the app at `http://localhost:3000`

## Troubleshooting

### 404 on API Requests

**Problem**: Requests to `/api/vms` return 404

**Solutions**:
- Check `API_URL` is correct and backend is running
- Verify backend is accessible: `curl http://localhost:3001/health`
- Check proxy logs for connection errors

### Static Files Not Found

**Problem**: Requests to `/` return 404 or Cannot GET /

**Solutions**:
- Verify `public/index.html` exists
- Build and deploy client: `cd .. && ./build`
- Check file permissions on `public/` directory

### Blank Page After Load

**Problem**: Page loads but shows blank screen

**Solutions**:
- Check browser console for JavaScript errors
- Verify Vite build completed successfully
- Check `public/assets/` contains JS/CSS bundles
- Inspect network tab for failed asset requests

### ECONNREFUSED

**Problem**: Proxy cannot connect to backend

**Solutions**:
- Verify backend is running: `curl http://localhost:3001/health`
- Check `API_URL` environment variable
- Ensure backend port matches `API_URL` (default 3001)
